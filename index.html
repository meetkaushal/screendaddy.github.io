Privacy

Your data lives on
your machine. Full stop.

There is no server, no analytics SDK, no account, no telemetry. ScreenDaddy was built with the assumption that your screen time data is sensitive — because it is.

🏠
100% local storage

All events are written to a local SQLite database on your Mac. Nothing leaves your machine unless you explicitly export it.

🚫
No cloud, no account

There is no sign-up, no sync service, no third-party SDK. ScreenDaddy has no internet dependency whatsoever.

👁
Window titles only

The Screen Recording permission is used only to read active window titles. ScreenDaddy never captures pixels, never takes screenshots.

FAQ

Common questions

Does ScreenDaddy take screenshots?

No. ScreenDaddy uses the Screen Recording permission only to read the title of the active window (e.g. "index.js — VS Code"). It never captures your screen, your pixels, or any visual content.

Where is my data stored?

Locally, in a SQLite database in your Application Support folder. You can find it at ~/Library/Application Support/ScreenDaddy/. Nothing is sent to any server.

What does the dashboard URL look like?

ScreenDaddy serves its dashboard at http://localhost:6769. Open it in any browser, or click "Open Dashboard" from the menu bar icon. It only works when ScreenDaddy is running.

What macOS versions are supported?

ScreenDaddy requires macOS 12 Monterey or later, on both Apple Silicon and Intel Macs.

What is the "Daddy password" for?

It's an optional password that locks the ability to quit the app or change settings. It's useful if you want to hold yourself accountable, or if you're setting up ScreenDaddy for a family member and want to prevent easy circumvention.

Can I use the API with my own scripts?

Yes. ScreenDaddy exposes a local REST API at http://localhost:6769/api/. Endpoints include /api/events, /api/timers, and /api/topApps. You can query them from any script, Shortcuts workflow, or tool that can make HTTP requests.
